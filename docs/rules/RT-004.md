# RT-004: Deep arithmetic chain without intermediate constraints

**Severity:** info | **Fixable:** manual | **Since:** 0.1.0

## What it detects

A `constrain_bits` instruction at the end of an arithmetic chain (8+ consecutive `add`/`mul`/`neg` operations) with no intermediate `constrain_bits` to force normalization.

## Why it matters

JS field arithmetic (`addField`, `subField`, `mulField` in `compact-runtime`) assumes inputs are in `[0, FIELD_MODULUS)` and uses single-reduction shortcuts. For example, `addField` adds two values and subtracts the modulus once if the result exceeds it. This works for single operations but may fail for long chains where intermediate values accumulate beyond `2 * FIELD_MODULUS`.

ZKIR uses native `Fr` arithmetic which handles arbitrary-depth chains correctly with full modular reduction at each step.

Severity is **info** because the JS shortcut works correctly for realistic chain depths in most contracts. It would require a very specific sequence of large values to trigger an actual overflow.

## Examples

### Long arithmetic chain (flagged)

```compact
pragma language_version >= 0.22.0;

export pure circuit polynomial_eval(
  x: Field,
  c0: Field, c1: Field, c2: Field,
  c3: Field, c4: Field, c5: Field,
  c6: Field, c7: Field, c8: Field
): Field {
  // 8+ chained additions without intermediate constraints
  const result = c0
    + c1 * x
    + c2 * x * x
    + c3 * x * x * x
    + c4 * x * x * x * x
    + c5 * x * x * x * x * x
    + c6 * x * x * x * x * x * x
    + c7 * x * x * x * x * x * x * x
    + c8 * x * x * x * x * x * x * x * x;
  return result as Field;
}
```

### Fixed: break into smaller chains with intermediate casts

```compact
pragma language_version >= 0.22.0;

export pure circuit polynomial_eval(
  x: Field,
  c0: Field, c1: Field, c2: Field,
  c3: Field, c4: Field, c5: Field,
  c6: Field, c7: Field, c8: Field
): Field {
  // Break into groups — constrain_bits at each cast forces normalization
  const low = (c0 + c1 * x + c2 * x * x + c3 * x * x * x) as Field;
  const x4 = x * x * x * x;
  const high = (c4 * x4 + c5 * x4 * x + c6 * x4 * x * x
    + c7 * x4 * x * x * x + c8 * x4 * x * x * x * x) as Field;
  return low + high;
}
```

## Technical details

The JS runtime's `addField` (`built-ins.ts`) computes `a + b` and subtracts `FIELD_MODULUS` once if the result exceeds it. This is correct for a single addition of two normalized values. But after 8+ chained additions, intermediate results may exceed `2 * FIELD_MODULUS`, and a single subtraction no longer normalizes correctly.
