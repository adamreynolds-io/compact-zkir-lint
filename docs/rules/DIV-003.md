# DIV-003: div_mod_power_of_two in conditional branch

**Severity:** warn | **Fixable:** manual | **Since:** 0.1.0

## What it detects

`div_mod_power_of_two` instruction operating on a value from a guarded region. This instruction performs `Field -> Bytes` conversion by decomposing a field element into quotient and remainder via bit extraction.

## Why it matters

`div_mod_power_of_two` extracts bits from field elements to decompose them. When the branch guard is false, the input defaults to 0, which decomposes safely (0 divided by any power of two = quotient 0, remainder 0). However, if the input mixes guarded and unguarded values through arithmetic, the result may be non-zero and fail bit extraction constraints.

Severity is **warn** rather than error because the zero default is safe in most cases.

## Examples

### Field-to-bytes inside branch (may break)

```compact
pragma language_version >= 0.22.0;

ledger stored: Bytes<32>;

export circuit store_encoded(val: Field, should_store: Boolean): [] {
  if (should_store) {
    stored = field_to_bytes(val);  // div_mod_power_of_two fires unconditionally
  } else {
    stored = pad(0, 32);
  }
  return [];
}
```

### Fixed: convert before the branch

```compact
pragma language_version >= 0.22.0;

ledger stored: Bytes<32>;

export circuit store_encoded(val: Field, should_store: Boolean): [] {
  const encoded = field_to_bytes(val);
  stored = should_store ? encoded : pad(0, 32);
  return [];
}
```

## Validation contract

[`tests/fixtures/compact/div-003-field-to-bytes-in-branch.compact`](../../tests/fixtures/compact/div-003-field-to-bytes-in-branch.compact) — compiles with `compact compile +0.30.0` and triggers DIV-003. The compiled ZKIR is at [`tests/fixtures/compiled/div-003-field-to-bytes-in-branch--conditional_extract.zkir`](../../tests/fixtures/compiled/div-003-field-to-bytes-in-branch--conditional_extract.zkir).

The contract reads a ledger `Counter`, casts it through `counter as Field as Bytes<32>` inside a branch, and feeds it into `persistentHash`. A direct `Field` parameter cast via `as Bytes<N>` generates `constrain_bits`, not `div_mod_power_of_two` — the instruction is only emitted when the value comes from a guarded `public_input` (ledger read).

## Compiler context

- **Root cause:** Compiler emits `div_mod_power_of_two` inside guarded regions without respecting the branch test
- **Compiler FIXME:** `passes.ss:9350` — "zkir field->bytes needs to respect test"
- **Tracking issue:** [compact#226](https://github.com/LFDT-Minokawa/compact/issues/226)
- **Affected versions:** 0.28.0+
