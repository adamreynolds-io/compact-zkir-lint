# DIV-004: assert on branch-local value

**Severity:** warn | **Fixable:** manual | **Since:** 0.1.0

## What it detects

`assert` instruction where the condition variable is derived from guarded computation (inside a conditional branch) and has not passed through a `cond_select` merge point.

## Why it matters

In JS, `assert` inside an `if/else` only executes when the branch is taken. In ZK, the assert fires unconditionally. If the dead branch produces a false condition value, the assert fails even though the branch is logically unreachable.

The proof server returns:

```
Failed direct assertion
```

Severity is **warn** because the dead-branch condition may happen to evaluate as true (e.g., if it defaults to 0 and the assert checks `value >= 0`). Whether it breaks depends on the specific assertion and input values.

## Examples

### Assert inside conditional (breaks)

```compact
pragma language_version >= 0.22.0;

ledger balance: Uint<64>;

export circuit withdraw(amount: Uint<64>, is_premium: Boolean): [] {
  if (is_premium) {
    // When is_premium is false, the dead branch still evaluates this assert.
    // If balance < amount in the dead branch, the assert fires.
    assert(balance >= amount, "Insufficient balance");
    balance = disclose(balance - amount) as Uint<64>;
  } else {
    assert(balance >= amount * 2, "Need 2x balance for standard users");
    balance = disclose(balance - amount * 2) as Uint<64>;
  }
  return [];
}
```

### Fixed: assert before the branch, or guard the condition

```compact
pragma language_version >= 0.22.0;

ledger balance: Uint<64>;

export circuit withdraw(amount: Uint<64>, is_premium: Boolean): [] {
  const cost = is_premium ? amount : amount * 2;
  assert(balance >= cost, "Insufficient balance");
  balance = disclose(balance - cost) as Uint<64>;
  return [];
}
```

## Validation contract

[`tests/fixtures/compact/div-004-assert-in-branch.compact`](../../tests/fixtures/compact/div-004-assert-in-branch.compact) — compiles with `compact compile +0.30.0` and lints **clean**. The compiled ZKIR is at [`tests/fixtures/compiled/div-004-assert-in-branch--guarded_assert.zkir`](../../tests/fixtures/compiled/div-004-assert-in-branch--guarded_assert.zkir).

Compiler 0.30.0 correctly handles this pattern by emitting `cond_select(flag, cond, true)` before the `assert` — the dead branch always asserts `true`. This rule serves as a regression detector: if a future compiler version drops the `cond_select` wrapper, the test will catch it.

## Compiler context

- **Root cause:** Compiler emits `assert` inside guarded regions without conditioning on the branch guard
- **Compiler FIXME:** `passes.ss:9655` — "are all of these okay if inputs are undefined?"
- **Tracking issue:** [compact#226](https://github.com/LFDT-Minokawa/compact/issues/226)
- **Affected versions:** 0.28.0+
