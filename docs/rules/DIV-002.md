# DIV-002: reconstitute_field in conditional branch

**Severity:** error | **Fixable:** manual | **Since:** 0.1.0

## What it detects

`reconstitute_field` instruction with operands (`divisor`, `modulus`) from a guarded region. This instruction performs `Bytes -> Field` conversion and emits internal constraints on its operands.

## Why it matters

`reconstitute_field` internally constrains its operands to validate the byte-to-field reconstruction. In a conditional branch, these constraints fire unconditionally on dead-branch values (which default to 0). The internal constraint validation may reject these default values, causing proof failure.

## Examples

### Bytes-to-field conversion inside branch (breaks)

```compact
pragma language_version >= 0.22.0;

ledger stored: Field;

export circuit store_converted(raw: Bytes<32>, use_raw: Boolean): [] {
  if (use_raw) {
    stored = bytes_to_field(raw);  // reconstitute_field fires even when use_raw is false
  } else {
    stored = 0 as Field;
  }
  return [];
}
```

### Fixed: convert before the branch

```compact
pragma language_version >= 0.22.0;

ledger stored: Field;

export circuit store_converted(raw: Bytes<32>, use_raw: Boolean): [] {
  const converted = bytes_to_field(raw);
  stored = use_raw ? converted : 0 as Field;
  return [];
}
```

## Compiler context

- **Root cause:** Compiler emits `reconstitute_field` inside guarded regions without respecting the branch test
- **Compiler FIXME:** `passes.ss:9671` — "zkir bytes->field needs to respect test"
