# DIV-001: constrain_bits in conditional branch

**Severity:** error | **Fixable:** manual | **Since:** 0.1.0

## What it detects

`constrain_bits(var, bits)` operating on an arithmetic result inside a guarded region (conditional branch), where the target variable has not passed through a `cond_select` merge point and is not guaranteed to be zero when the branch guard is false.

## Why it matters

In ZK circuits, both branches of an `if/else` execute unconditionally. The compiler emits `constrain_bits` for `as Uint<N>` casts, which validates that a value fits in N bits. When the branch is not taken, dead-branch arithmetic may produce out-of-range intermediate values that fail this constraint.

The proof server returns:

```
BadInput("Bit bound failed: 000000000000000001 is not 64-bit")
```

This is the most common divergence pattern. It affects **every contract** using `as Uint<N>` inside `if/else`, including contracts depending on OpenZeppelin's `Uint128` math library.

## Examples

### Narrowing cast inside branch (breaks)

From [`tests/fixtures/compact/cast-in-branch.compact`](../../tests/fixtures/compact/cast-in-branch.compact) (verified in [`tests/e2e.test.ts`](../../tests/e2e.test.ts)):

```compact
pragma language_version >= 0.22.0;

ledger value: Uint<8>;

export circuit set_clamped(x: Uint<32>): [] {
  if (disclose(x) < 256) {
    value = disclose(x) as Uint<8>;  // constrain_bits(8) fires even when x >= 256
  } else {
    value = 255;
  }
  return [];
}
```

When `x >= 256`, the `else` branch is taken in JS, but ZK still evaluates `x as Uint<8>` — and `x` doesn't fit in 8 bits.

### Fixed: cast after branch merge

```compact
pragma language_version >= 0.22.0;

ledger value: Uint<8>;

export circuit set_clamped(x: Uint<32>): [] {
  const clamped = disclose(x) < 256 ? disclose(x) : 255;
  value = clamped as Uint<8>;  // constrain_bits on SELECTED value — always <= 255
  return [];
}
```

### Real-world: OpenZeppelin Uint128.subU128

Workaround by the [OpenZeppelin](https://github.com/OpenZeppelin) team in [midnight-apps#309](https://github.com/OpenZeppelin/midnight-apps/pull/309). The `subU128` circuit in their Compact math library hit this pattern in production via LunarSwap ([compact#250](https://github.com/LFDT-Minokawa/compact/issues/250)).

```compact
// BEFORE (broken — Compact 0.30.0): as Uint<64> in else branch
export pure circuit subU128(a: U128, b: U128): U128 {
  const highDiff = a.high - b.high;
  if (a.low >= b.low) {
    return U128 { low: a.low - b.low, high: highDiff };
  } else {
    // borrow: add MODULUS (2^64) to a.low before subtracting
    return U128 {
      low: (a.low + MODULUS() - b.low) as Uint<64>,  // DIV-001: fires when a.low >= b.low
      high: highDiff - 1
    };
  }
}

// AFTER (fixed): branchless, cast always safe
export pure circuit subU128(a: U128, b: U128): U128 {
  const borrow = a.low < b.low ? 1 : 0;
  const lowDiff = a.low + borrow * MODULUS() - b.low;
  const highDiff = a.high - b.high - borrow;
  return U128 { low: lowDiff as Uint<64>, high: highDiff };
}
```

See [Branchless Patterns](../guides/branchless-patterns.md) for more fix strategies.

## Compiler context

- **Root cause:** Compact compiler emits unguarded `ConstrainBits` for dead branches
- **Compiler FIXME:** `passes.ss:9676` — "zkir downcast-unsigned with safe? = #f"
- **Tracking issue:** [compact#226](https://github.com/LFDT-Minokawa/compact/issues/226)
- **Affected versions:** 0.28.0+
