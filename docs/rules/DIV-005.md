# DIV-005: constrain_eq in conditional branch

**Severity:** warn | **Fixable:** manual | **Since:** 0.1.0

## What it detects

`constrain_eq(a, b)` where one or both operands are from a guarded region and neither has passed through a `cond_select` merge point.

## Why it matters

`constrain_eq` fires unconditionally in ZK. If one operand is from a dead branch (defaulting to 0) and the other is a live value from the parent scope, the equality check fails because the values differ.

Severity is **warn** because if both operands are from the same guarded region and both default to 0, the equality check passes (0 == 0). It only breaks when operands have different provenance.

## Examples

### Equality constraint inside branch (breaks)

```compact
pragma language_version >= 0.22.0;

ledger expected_hash: Field;

export circuit verify_and_store(
  input: Field,
  hash: Field,
  is_verified: Boolean
): [] {
  if (is_verified) {
    // constrain_eq fires unconditionally; when is_verified is false,
    // dead-branch hash (0) != expected_hash
    assert(hash == expected_hash, "Hash mismatch");
    expected_hash = input;
  }
  return [];
}
```

### Fixed: check after branch merge

```compact
pragma language_version >= 0.22.0;

ledger expected_hash: Field;

export circuit verify_and_store(
  input: Field,
  hash: Field,
  is_verified: Boolean
): [] {
  // Guard the constraint: only require hash match when is_verified
  assert(!is_verified || hash == expected_hash, "Hash mismatch");
  expected_hash = is_verified ? input : expected_hash;
  return [];
}
```
