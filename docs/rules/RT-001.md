# RT-001: persistent_hash with guarded inputs

**Severity:** warn | **Fixable:** manual | **Since:** 0.1.0

## What it detects

`persistent_hash` instruction where one or more input variables are from a guarded region (conditional branch).

## Why it matters

ZKIR and JS handle hash inputs differently:

- **JS** (`compact-runtime`): hashes `AlignedValue` objects directly via `type.toValue()`
- **ZKIR** (`ir_vm.rs`): re-parses field elements through alignment, then converts to binary before hashing

When guarded inputs default to 0 in the dead branch, the alignment parsing in ZKIR may produce different binary than what JS hashed. This means the same circuit produces different hash outputs in JS vs ZK, causing proof validation to reject the transaction.

## Examples

### Hash inside conditional branch (may diverge)

```compact
pragma language_version >= 0.22.0;

ledger commitment: Field;

export circuit commit_if_ready(
  value: Uint<64>,
  nonce: Bytes<32>,
  is_ready: Boolean
): [] {
  if (is_ready) {
    // persistent_hash inputs are guarded — ZKIR re-parses them through alignment
    commitment = persistent_hash(value, nonce);
  }
  return [];
}
```

### Fixed: hash before the branch

```compact
pragma language_version >= 0.22.0;

ledger commitment: Field;

export circuit commit_if_ready(
  value: Uint<64>,
  nonce: Bytes<32>,
  is_ready: Boolean
): [] {
  const hash = persistent_hash(value, nonce);
  commitment = is_ready ? hash : commitment;
  return [];
}
```

## Validation contract

[`tests/fixtures/compact/rt-001-persistent-hash-guarded.compact`](../../tests/fixtures/compact/rt-001-persistent-hash-guarded.compact) — compiles with `compact compile +0.30.0` and triggers RT-001. The compiled ZKIR is at [`tests/fixtures/compiled/rt-001-persistent-hash-guarded--conditional_hash.zkir`](../../tests/fixtures/compiled/rt-001-persistent-hash-guarded--conditional_hash.zkir).

The contract calls `persistentHash` with a `Field` witness read inside a branch. The witness must come from a `witness` function (not a circuit parameter) so the compiler emits `private_input` with a guard.

## Technical details

The divergence occurs in `ir_vm.rs` at the `resolve_operand_bits()` call during ZKIR preprocessing. Field elements are parsed back into their aligned representation, then serialized to binary for the hash function. The JS runtime skips this round-trip — it hashes the `AlignedValue` directly from `type.toValue()`.
